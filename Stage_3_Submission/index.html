Stage 1, Work session 1 - HTML and Tags

Web pages are text documents created using a text editor using a special language called HTML. Browser software on your PC is able to display these pages as intended by interpreting the words and codes within the text document after downloading them over the internet from a server where the pages are stored. There are a huge number of servers and web pages around the world and these together comprise the "world wide web".

Pages of HTML comprise two main sections, the head and the body. The head section contains information about the page such as its title and what CSS or Javascript is to be used, whereas the body section contains the content to be displayed.

The HTML language comprises elements which contain tags with attributes and text.

Some example tags are the anchor tag <a>, the div tag <div> and the bold tag <b>.

Tags can be inline or block, inline are used just for text whereas block tags create an invisible box around the text which can be resized and have other properties assigned to it.

Special text editing software exists that make writing HTML and other programming languages easier by pre-empting the words you are typing and displaying the code you have written in special colours and formatting that make it easier to read and understand. Sublime Text is a good example of such a text editor.

Stage 1, Work session 2 - Document Object Model (DOM)

The structure of an HTML webpage is stored in the Document Object Model (DOM) and is comprised of all the elements that exist in the HTML page. You can use the developer tools in a browser such as Google Chrome to see the structure of a web page in action and make changes to see what each does. This can be a great way of learning. Also it can be very educational to actually draw the structure of a web page diagrammatically with a pencil and paper like in the example below.

HTML controls the structure of a web page, but the style that you see is controlled by CSS. Applying a 'class' to a div tag can be useful to give it a name or tell it what style to use.

All web pages are comprised of a number of boxes, some of the boxes can be inside other boxes.

Stage 1, Work session 3 - CSS

CSS determines the presentation of HTML documents and is used to change the appearance of the web page without the need to change the content or structure.

One can define the content characteristics of parts of the web page either by HTML element selectors such as h1, p etc or by class selectors that you can define yourself.

CSS text should be placed in a separate file with a file extension of .css. In your HTML you tell the page what css file to use and where it is stored.

Use of CSS and careful consideration to the structure of your HTML code leads to clean and easy to understand web pages and avoids the need to repeat unnecessary bits of code.

HTML and CSS can be checked by using online validation tools such as can be found here for HTML and here for CSS at W3C.

Stage 2, Work session 1 - Variables and Strings

When writing computer programs, values can be assigned to a location which is called assignment. The assignment points to a location where the value is stored. You assign a value to a variable as follows, a=4. Variables can have a different value assigned to them which replaces the prior value stored at the location. Variables can have operations performed on them such as addition and multiplication. For example, using the assignment of the variable a above the following would happen,
print 5 * a
(would ouput 20).

The value assigned to a variable can also be a string. Strings can also have operations performed on them, for example a = "mike", print 4 * a (would output mikemikemikemike). Strings can be sliced i.e. taking a part of the string in order to perform some other operation on it.

Strings are sequences of characters and as such can be iterated through. Strings (along with Tuples) are immutable, meaning that their values cannot be changed within the assignment but a new string can be created. Unlike Lists and Dictionaries which allow element substitution within the existing construct.

Stage 2, Work session 2 - Using and Defining Functions

You can use a method called find in order to find a character or string within a string. It returns the position within the string that the search string or character was found. If it was not found, -1 is returned.

You can define your own functions that can be called from within your code. The function executes and then returns to your code which then continues from where it left off. Functions can return a value to be used in the future processing of your code. An example of a function is below,

def sum(a,b):
a = a + b
return a
which could be called as follows,
print sum (2, 2)
the output being 4, obviously!

Parameters are usually passed in to a function to be used in some way. Different values may be passed in an iterative process such as working through a list doing the same thing to each element. A function can call another function, and so on. This helps reduce repetition in your code.

Functions can return a value to the main code to be evaluated further. If there is no return, no value is returned to the code that called the function.

There are many system functions available within Python that can be used.

Stage 2, Work session 3 - Decisions and Repetition

In order to control the flow of your code you need to be able to make decisions and do one thing for one outcome and another otherwise.

The 'if' statement allows you to perform comparisons resulting in an 'if this is so, then do this, otherwise do something else' scenario. The outcome of a comparison is always either 'True' or 'False' e.g. if comparison is true/correct, then return 'True'. If it is not true/incorrect, then return 'False'.

It is also possible to create 'if' statements that contain an extra step 'else' that says if the condition is not met then do something else, surprisingly.

If statements can be nested in order to perform more complex comparisons.

Stage 2, Work session 4 - Lists and For loops

Loops can be created to cycle through the values contained in some kind of list. Different kinds of loops exist for different scenarios. For example, 'while' loops will continue to execute while a condition remains 'True', whereas a 'for' loop will execute a certain number of times based on some criteria, such as counting through the actual number of values in a list.

Lists can contain all different types of data and can even contain other lists. It is a common requirement in coding to need to cycle through all the values in a list or all the values that meet a certain criteria.

It is also common to create a 'list' from a list in order to prepare the values to be processed further because they meet the reason or purpose we are looking for, such as all words beginning with 'U' for example.

The use of 'for' loops can be very useful for going through each value in a list. The structure of a 'for' loop can take the form as follows. For x(name) in list(List): Do something over and over again until there are no more elements and then branch off to the next instruction to be followed.

Lists are mutable and Python contains list operations that are built in methods which are very useful. Examples would be .append which as it implies appends an element to the list, or .sort which (again as its name suggests) sorts the elements in a list in to an order such as ascending.

Stage 3, Work session 1 - Inroduction to Object Oriented Programming

Object oriented programming allows you to create consistency and the re-use of the code that you produce. Python is an object oriented programming language because it gives you the constructs required to create re-usable code such as class definitions and inheritance (more on inheritance later).
This introduces the concept of abstraction, which simply means using bits of code previously produced without the need to know how it does it, we just re-use it. You can import other programs to re-use in your own code, but make sure the paths necessary for proper operation are in place or make sure all of the code is in the same directory. Examples of modules experienced so far include OS, Time & webbroswer.

Without this ability to re-use code and share definitions larger programming projects would require you to craft many more lines of bascially very similar code to achieve the same result. Larger code takes longer to write and is harder to maintain, especially if you are not the one who wrote it intially.

A module can be created that just does one thing. And if you need to do that same thing another time you can just import the whole module in to your new project and re-use it in its entirety or just call upon a part of it.

A library is a collection of modules that have been assembled in a group because of similarity or purpose. Examples of Python libraries are NumPy (great with numbers) and PyQt (a graphical GUI toolkit).

Stage 3, Work session 2 - Object, Class & Instance

Objects are just things. Most things in Python are objects. When we require another object that is basically similar to one we have already created, we call it an instance of the other object. Classes are object instance creating engines. They allow you to create multiple similar objects without having to completely re-do them every time. A class can be considered as a blueprint for something you will need to build several times. Classes create an instance of an object by using an internal function called __init__.

Some system functions behave very much like classes in that they create instances just like classes do. For example, the function 'file' within module OS is used to create an instance of an object that includes the contents of a file.

Stage 3, Work session 3 - Constructor, Self & Inheritance

Classes are used to create or define new instances based on the class definition previously created. Below is an example,

variable = module.ClassName():

which might equate in a real program to something like,

salmon = species.Fish():

Salmon is the name of the instance you are creating. This is being instantiated by using a Class called Fish which can be found in a file or module called species.

When a class is created a special function is defined called __init__ which is also known as a constructor. The use of double underscores either side of a function name is a way for Python to let us know that they are reserved names and therefore should not be used for other purposes.

The first attribute in the definition __init__ is always something called self. Self can be considered either as itself or the name of the instance you are creating.

Class variables (i.e. variables that are declared in the class definition) can be used by all instances of the class that are created. Convention is to name class variables with all upper case characters.

There are some pre-defined class variables such as __doc__, __main__ & __module__. For example, __doc__ will be assigned the content of any docstring included in the class definiton.

Inheritance is the sytematic re-use of code. In OOP this means the re-use of one class within another class. One class is said to have inherited the code of another class and is illustrated as follows,

class Child(Parent):
       def __init__(self, last_name, eye_color, number_of_toys):

A method inherited from another class can be overridden if necessary.
